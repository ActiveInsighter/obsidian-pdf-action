# 简洁灰色主题代码高亮测试

> [!NOTE] 测试目标
> 使用 `theme: obsidian-inspired` 检查原来的简单灰色系主题：代码块背景保持 `#f7f7f8`，无边框，只调整高亮颜色，让整体更柔和、更协调。

这是一份紧凑风格测试。正文间距保持偏紧，标题、表格和列表不明显放松，适合做复习资料和快速阅读版 PDF。

## 1. 代码块视觉要求

- 背景：保持浅灰 `#f7f7f8`。
- 边框：保持无边框。
- 颜色：低饱和、但能区分关键字、函数、字符串、属性和注释。
- 整体：不要像 IDE 那样花哨，要像文档里的代码示例。

## 2. JavaScript

```javascript
const theme = 'obsidian-inspired';
const codeBlock = {
  background: '#f7f7f8',
  border: 0,
  palette: 'compact-neutral-gray',
};

function renderPdf(input, output) {
  if (!input.endsWith('.md')) {
    throw new Error('Only Markdown files are supported');
  }

  return {
    input,
    output,
    theme,
    ok: true,
  };
}

console.log(renderPdf('note.md', 'note.pdf'));
```

## 3. Python

```python
from dataclasses import dataclass
from pathlib import Path

@dataclass
class PdfTheme:
    name: str
    code_background: str = '#f7f7f8'
    borderless_code: bool = True


def collect_markdown_files(root: Path) -> list[Path]:
    return sorted(root.glob('*.md'))


theme = PdfTheme(name='obsidian-inspired')
print(theme)
print(collect_markdown_files(Path('md')))
```

  AND status = 'success'
ORDER BY finished_at DESC

## 6. 公式兼容检查

行内公式：\( \left|x-1\right| \)、\( x_{i,j}^{(k+1)} \)、\( \left\lVert A\boldsymbol{x}-\boldsymbol{b}\right\rVert_2 \)。

块级公式：

\[
\left|\sum_{i=1}^{n}a_i b_i\right|
\le
\left(\sum_{i=1}^{n}a_i^2\right)^{1/2}
\left(\sum_{i=1}^{n}b_i^2\right)^{1/2}
\]

\[
\begin{aligned}
J(\theta)
&= \frac{1}{2m}\sum_{i=1}^{m}\left(h_\theta(x^{(i)})-y^{(i)}\right)^2 \\
\frac{\partial J}{\partial \theta_j}
&= \frac{1}{m}\sum_{i=1}^{m}\left(h_\theta(x^{(i)})-y^{(i)}\right)x_j^{(i)}
\end{aligned}
\]

| 检查项 | 示例 | 目标 |
|---|---|---|
| 绝对值 | \( \left|x_i-y_i\right| \) | 竖线稳定 |
| 角标 | \( x_{i,j}^{(k+1)} \) | 不挤压正文 |
| 范数 | \( \left\lVert \boldsymbol{x}\right\rVert_2 \) | 双竖线协调 |
